c3spec Bootstrap T2 — Lightweight Feature

Proposed: 2026-05-25  ·  Repo: shwcsmack/c3spec  ·  Change: tier2-c3spec-bootstrap

Why

Pain: The project relies on @fission-ai/openspec — a generic, third-party tool with telemetry we don't control, a binary name that carries no brand identity, and an init command that scaffolds a generic structure instead of the opinionated Code 3 Dev workflow.

Why now: The workflow redesign (tiered routing, custom subagent orchestration, memory system) is complete in open-brain. The next step is owning the full stack — the CLI, the schema, and eventually the plugin — so future projects can be bootstrapped from a single c3spec init instead of manually copying files.

Expected gain: A fully branded, telemetry-free CLI that dogfoods itself. Running c3spec init in any new project will scaffold the entire tiered workflow, memory structure, and custom skills — zero manual setup.

New c3spec memory Command

A new subcommand for managing the c3spec/memory/ system. Three operations:

  • c3spec memory list — print MEMORY.md index with file sizes and dates
  • c3spec memory add <category> <slug> — scaffold a new memory file with correct frontmatter and add index entry
  • c3spec memory promote <slug> — mark a retro candidate as promoted and verify the memory file exists

Capabilities

memory-management

New capability: c3spec memory subcommand for listing, adding, and promoting project memory entries. Operates on the c3spec/memory/ directory and MEMORY.md index.

c3spec-init (modified)

The existing init command is extended to scaffold the full Code 3 Dev workflow structure — superpowers-bridge schema, tiered routing skills, CLAUDE.md fragment, and memory directory — as the default profile.

Scope

In scope

  • Package rename (c3spec binary, package name, all internal references)
  • Workflow directory rename: openspec/ → c3spec/ (CLI path constant + default config)
  • Strip PostHog telemetry entirely
  • Custom init default profile (superpowers-bridge + memory + skills)
  • c3spec memory subcommand (list, add, promote)
  • README and branding updates
  • Private publish setup (installable from GitHub)

Out of scope

  • New spec validation logic
  • Upstream OpenSpec feature parity
  • Plugin packaging for multi-project distribution
  • Public npm registry publishing
  • Windows CI (carry forward from upstream)
