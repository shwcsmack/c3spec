Tier 2 Proposal · 2026-05-26

Codify Workflow Routing as a Spec

Add a new source-of-truth behavioral spec for c3spec's tier routing contract: the c3spec-start front door, T1/T2/T3 classification, canonical tier skills, required workflow phases, and the existing enforcement surfaces that keep the generated host artifacts aligned.

Why

The tier workflow is one of c3spec's core product contracts, but today it is spread across CLAUDE.md, AGENTS.md, generated instruction source, and six skill files. Other stable behaviors in the repo are documented under c3spec/specs/; routing is not. Capturing it as a behavior-first spec gives future workflow changes a shared target and reduces drift between instructions, skills, tests, and generated host artifacts.

What Changes

Area From To Impact
Workflow routing contract Documented in instructions and skill prose only. Documented as workflow-routing under c3spec/specs/. New spec
Tier set and entry path Implied by c3spec-start and host instruction blocks. Specified as a stable contract: all work enters through c3spec-start, then routes to T1/T2/T3. Clarifies behavior
Canonical skills and agents Enforced by constants/tests, not described in a workflow spec. Spec names the canonical skill and review-agent surface at behavior level. No code change
Spec/test enforcement No project-wide requirement-to-test coverage convention exists. Explicitly out of scope for this change; captured as IDEAS.md #15. Deferred

New Capability

workflow-routing: Defines the c3spec tier workflow contract, including the front-door interview, routing classifier behavior, tier-specific workflow shape, canonical skill/agent surfaces, and expected enforcement boundaries.
