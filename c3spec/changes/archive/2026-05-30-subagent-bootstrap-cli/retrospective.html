Retrospective: subagent-bootstrap-cli

1. Evidence

  • Implemented new command: c3spec subagent bootstrap --change <id> [--json]
  • Added bootstrap engine: required checks (runtime, artifacts, roles) + informational memory
  • Added distinct exit-code behavior and JSON output contract
  • Added tests: command behavior + skill-gating contract
  • Verification: full test suite passed (78 files, 1232 tests)

2. What worked

  • Design decisions from interview translated cleanly into command contract.
  • Stable check IDs and categorized output made testing straightforward.
  • Fast feedback loop via targeted tests before full suite run.

3. What did not work

  • Runtime validation is still contract-surface based, not host-process introspection.
  • Dispatch integration currently relies on updated skill contracts rather than deeper runtime hooks.

4. Workflow/process improvements

  • Keep a reusable exit-code/failure taxonomy module for future gate commands.
  • Add a shared helper for tier/artifact derivation to avoid duplicated lifecycle logic.

5. Generalizable learning

For workflow-gating commands, separating required vs informational checks with stable IDs and distinct exit codes significantly improves reliability and automation ergonomics.
