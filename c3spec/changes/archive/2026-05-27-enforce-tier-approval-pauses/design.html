Design: enforce-tier-approval-pauses

Define one lifecycle-owned policy for pause gates, HTML handoff ordering, fast-forward behavior, and approval interpretation across T1/T2/T3.

Core Decisions

DecisionDesign ChoiceReason
Source of truth All pause/handoff policy defined in c3spec-tier-lifecycle. Prevents drift from repeated local rules in tier skills.
Tier 3 HTML rule Require HTML for proposal, design, retrospective. Tier 3 is highest-risk planning surface and benefits from explicit review gates.
Tier 1/2 HTML rule Optional HTML companions; when used enforce ordered handoff. Keeps lower tiers lightweight without losing consistency.
Non-pausing artifacts tasks.md and plan.md are non-pausing. Reduces unnecessary interaction overhead.
Verification gate verify.md non-blocking unless verification fails. Keeps momentum while preserving quality stop on failure.
Fast-forward semantics Skip pauses and skip HTML generation, default through retrospective, stop after retro for review. Gives explicit acceleration path while keeping archive review safety.
Approval interpretation Accept clear natural-language affirmative approvals. More human-friendly than brittle keyword matching.

Contract Model

IF artifact has HTML companion and HTML is in effect:
generate HTML -> show review path -> wait for approval (unless fast forward) -> save markdown durable record

IF artifact is markdown-only:
save markdown -> continue (pause only when lifecycle marks artifact as gate and not fast forward)

IF fast forward active:
skip approval pauses + skip HTML generation -> continue through retrospective -> stop for retro review before archive

Risks And Mitigations

Risk: Divergent prose between lifecycle and tier skills

Mitigation: lifecycle-first edits, then consumer updates in same change, with focused tests asserting references and key clauses.

Risk: Ambiguous fast-forward state when resuming

Mitigation: define deterministic default behavior in resume skill and ensure stop-after-retro is explicit.

Risk: Approval interpretation too loose

Mitigation: define examples of clear affirmatives in skill prose and keep non-affirmative replies as pending-review state.
