Proposal: enforce-tier-approval-pauses

Tier 3 Date: 2026-05-27 Branch: feat/enforce-tier-approval-pauses

Why

Tier workflows currently have inconsistent pause behavior and inconsistent HTML-to-markdown handoff timing. This reduces user trust in approvals, causes artifact drift, and weakens fresh-context resumability.

The lifecycle contract already exists as the intended single source of truth, so this change formalizes and enforces the pause and handoff behavior there and aligns all tier/resume consumers.

What Changes

Area Before After
Pause behavior Partially duplicated across tier skills; inconsistent gate behavior. Canonicalized in c3spec-tier-lifecycle and consumed by tier/resume flows.
tasks.md / plan.md Could act like pause points in practice. Explicitly non-pausing artifacts.
verify.md Pause semantics ambiguous. Non-blocking unless verification fails.
HTML review usage Mixed expectations per tier/artifact. Tier 3 requires HTML for proposal/design/retrospective; Tier 1/2 optional.
Handoff ordering Markdown could be saved before review in some flows. When HTML is used: HTML -> approval -> markdown durable record.
fast forward No canonical cross-tier behavior. Skips pauses and HTML generation, defaults through retrospective, stops after retro for review.
Approval parsing Keyword assumptions varied. Clear natural-language affirmative acceptance.

Impact

  • Affected skills: c3spec-tier-lifecycle, c3spec-tier1-fix, c3spec-tier2-feature, c3spec-tier3-full, c3spec-continue-change.
  • Affected specs likely include workflow-routing and canonical-skills.
  • Focused tests under test/specs/ will be updated/added to lock pause and handoff rules.
  • No intended behavior change to unrelated CLI command surfaces.
