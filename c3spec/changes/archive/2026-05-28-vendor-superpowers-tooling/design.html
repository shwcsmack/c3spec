Design: vendor-superpowers-tooling

Decisions

  1. Vendor-now set: replace direct dependencies on superpowers:using-git-worktrees and superpowers:finishing-a-development-branch.
  2. Naming: use local canonical skills c3spec-using-git-worktrees and c3spec-finishing-development-branch.
  3. Nested dependency audit: keep explicit map of transitive skills (brainstorming, writing-plans, subagent-driven-development, TDD, requesting-code-review) for future adoption.

Architecture/flow

Tier/archive skills invoke local c3spec skills. Local skills encapsulate command procedures and remove hard dependency on external plugin naming.

Contracts

  • Tier workflows must remain behaviorally equivalent at worktree setup and branch finish points.
  • Skill references in canonical files must no longer require external superpowers:* names for critical path execution.

Risks and mitigations

  • Risk: host artifact drift. Mitigation: run canonical/host generation tests.
  • Risk: semantic mismatch with old skill behavior. Mitigation: encode explicit steps in new local skills and verify with tests.

Rollout

Ship vendored critical skills now; track remaining skills in the change documentation as prioritized follow-up candidates.
