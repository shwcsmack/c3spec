## Why

`c3spec update` generates skill files from templates compiled into the CLI binary. Improving a skill requires a full npm publish plus a user CLI upgrade — two steps most users won't connect. Running `c3spec update` with an old CLI silently writes stale skill content. This change makes `c3spec update` fetch skill content directly from the GitHub repository, so improvements ship the moment they land on `main` regardless of which CLI version is installed.

## What Changes

**Skill template delivery**
- From: skill content is compiled into the CLI binary; `c3spec update` writes bundled templates
- To: `c3spec update` fetches skill content from `raw.githubusercontent.com/shwcsmack/c3spec/main/skills/<workflow>/SKILL.md`; falls back to bundled silently on network failure
- Reason: decouple skill improvements from CLI release cycle
- Impact: non-breaking; offline behavior preserved via fallback

**Skill content source of truth**
- From: TypeScript template functions in `src/core/templates/workflows/*.ts` are hand-edited
- To: raw `skills/<workflow>/SKILL.md` files are the source of truth; TypeScript templates are generated by codegen during build
- Reason: eliminate drift between skill files and bundled content
- Impact: developers edit `.md` files, never TypeScript templates directly

**Build pipeline**
- From: `node build.js` runs `tsc` only
- To: `node build.js` runs codegen (`scripts/generate-templates.ts`) then `tsc`
- Reason: keep generated TypeScript always in sync with source markdown
- Impact: build time increases slightly; CI gains a drift-detection step

## Capabilities

### New Capabilities
- `remote-skill-fetch`: fetching skill content from GitHub during `c3spec update`, with silent fallback to bundled templates on network failure
- `skill-template-codegen`: generating TypeScript skill template functions from raw `skills/*.md` source files as part of the build pipeline, with CI enforcement

### Modified Capabilities
- `cli-update`: update command now fetches remote skill content before writing to user projects

## Impact

- `src/core/update.ts`: fetch logic added to skill generation flow
- `src/core/shared/skill-generation.ts`: accepts fetched content alongside bundled templates
- `src/core/templates/workflows/*.ts`: become generated files, not hand-edited
- `skills/`: new top-level directory with one `SKILL.md` per workflow
- `scripts/generate-templates.ts`: new codegen script
- `build.js`: runs codegen step before `tsc`
- CI: new step to assert generated TypeScript matches source markdown
- No new npm dependencies required (Node.js built-in `fetch`, available ≥18)
