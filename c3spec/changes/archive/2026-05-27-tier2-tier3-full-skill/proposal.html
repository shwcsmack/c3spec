Tier 2 Lightweight Feature · May 26, 2026

Author a c3spec-tier3-full Skill

Add a first-class Tier 3 workflow skill so full c3spec changes follow explicit, repeatable instructions instead of being assembled ad hoc from scattered skill references.

Recommended scope: contained workflow capability

Why

Tier 1 and Tier 2 already have dedicated skill files that spell out pre-flight, planning, implementation, verification, retrospective, and finishing steps. Tier 3 is currently only described inline in c3spec-start, where Step 4 tells the agent to begin the full superpowers-bridge flow and choose the relevant skills.

Problem: Tier 3 is the most expensive workflow to get wrong, but it has the least explicit routing surface. That makes agent behavior depend on session memory, host quirks, and whichever related skills the agent happens to read.

What Changes

Before After
c3spec-start routes Tier 3 with a single inline sentence. c3spec-start routes Tier 3 to c3spec-tier3-full.
Agents infer the sequence across brainstorming, proposal, design, specs, tasks, plan, apply, verify, retro, and archive. The Tier 3 skill defines each stage, handoff, HTML checkpoint, and required approval gate.
Generated host artifacts may omit the new workflow unless the canonical host skill list is updated. The skill is added to the host-generation skill list and covered by skill-list tests.

New Capabilities

  • c3spec-tier3-full documents the canonical full workflow end to end.
  • Tier 3 planning artifacts get explicit HTML review checkpoints before markdown save.
  • Implementation handoff to c3spec-subagent-dev is defined with Tier 3 final review behavior.
  • Archive, retrospective, verification, and memory capture are described as required completion steps.

Impact

Area Expected Files Notes
Skill source skills/c3spec-tier3-full/SKILL.md Source-of-truth skill content used by template codegen.
Canonical agent skills .agents/skills/c3spec-tier3-full/SKILL.md, .agents/skills/c3spec-start/SKILL.md Host-facing canonical skills should include the new Tier 3 route.
Host generation src/core/host-generation/types.ts, src/core/host-generation/renderers/instructions.ts Register the Tier 3 skill and update injected routing guidance.
Tests test/core/init.test.ts, test/core/update.test.ts, related skill template tests Add or update assertions so the bundled skill list includes Tier 3.
Specs c3spec/specs/skill-template-codegen/spec.md or a workflow-routing spec if one exists/gets added Only update specs if implementation changes observable behavior beyond bundled skill coverage.

Acceptance Criteria

  • There is a dedicated c3spec-tier3-full skill with pre-flight, planning, apply, verify, retro, archive, finish, and anti-pattern sections.
  • c3spec-start no longer asks the agent to improvise the Tier 3 flow inline.
  • Host-generation skill lists and routing instructions include c3spec-tier3-full.
  • Tests cover the new skill being installed or updated through the existing host generation path.
  • pnpm test and TypeScript verification pass.

Main risk: keeping multiple skill sources in sync. The plan should prefer existing codegen over hand-maintained duplication wherever possible.
